# Contributing to the Framework

The framework is designed to grow through community contribution. 
Four types of contribution are welcome:

## 1. Domain Instantiations
Apply the framework to a new domain (water, transport, land-use, etc.).
Specify: decision class, module structure, artefact families, uncertain drivers.

## 2. Module Implementations
Implement a specified module (e.g., PyPSA regional module, OpenModelica 
site module). Must satisfy the interface contract of the module it implements.

## 3. Methodological Extensions
Extend the evaluation layer with new robustness metrics, scenario 
discovery approaches, or adaptive pathway formulations.

## 4. Corrections and Critiques
Identify errors or propose alternative approaches.

## Submission Process
1. Open a GitHub Discussion in the appropriate category
2. Use the contribution template for your category
3. The editorial process assesses compatibility with the thin-waist 
   architecture and the decision-first principle
4. Accepted contributions are incorporated with full provenance
